1. 지시어 (Directive)
    1) JSP 페이지를 자바(서블릿)코드로 변환하는데 필요한 정보를 JSP 엔진에 알려줌
        - 주로 스크립트 언어, 인코딩 방식 설정
        - <%@ 지시어 속성1="값1" 속성2="값2; 속성3=값3" ... %>
            - 지시어 종류
                1] page : JSP 페이지에 대한 정보 설정
                2] include : 외부파일을 현재 JSP 페이지에 포함시킴
                3] taglib : 표현 언어(EL)에서 사용할 자바 클래스, JSTL을 선언

    2) page 지시어 
        - import
            - 페이지에서 사용할 자바 패키지와 클래스를 지정함
        - errorPage
            - 해당 페이지에서 에러가 발생했을 때 에러발생 여부를 보여줄 페이지를 지정함.
        - isErrorPage
            - 해당 페이지에서 에러를 처리할지 여부를 지정함 ((default : false) => true 변환시 에러 처리)
        - trimDirectiveWhitespaces
            - page 지시어로 생긴 불필요한 공백 제거
        - buffer , autoFlush
            - 출력할 내용을 버퍼에 저장했다가 일정량이 되었을 때 전송하게 됨.
            - 버퍼를 사용함으로써 포워드(forward, 페이지전달)와 에러페이지 처리 가능
            - JSP가 생성한 결과는 일단 버퍼에 저장됨
                - buffer 기본값은 8kb  
                - autoFlush 기본값은 true : 버퍼가 채워지면 자동으로 플러시함
                                   false : 버퍼가 채워지면 예외를 발생시킴
                    - flush : 버퍼 안의 데이터를 목적지로 전송하고 버퍼를 비우는 작업을 말함.

    3) include 지시어
        - 다른 문서를 포함시키면 먼저 '파일 내용 그대로를 문서에 삽입'한 후
          '컴파일'이 진행됨. => 하나의 페이지가 됨.

2. 스크립트 요소 (Script Elements)
    1) jsp에서 자바 코드를 직접 작성할 수 있게 해줌.
        - 선언부 , 스크립틀릿, 표현식
    2) 선언부 (Declaration)
        - 스크립틀릿이나 표현식에서 사용할 멤버 변수나 메서드 선언함
        - <%!변수 , 메서드 등 선언 %>
    3) 스크립틀릿(scriptlet)
        - JSP 페이지가 요청을 받았을 때 실행되야 할 자바 코드를 작성하는 영역임
        - 서블릿으로 변환시 _jspService()메서드 내부에 그대로 기술됨
        - <% 자바 코드 %>
    4) 표현식 (Expression)
        - <%= 자바표현식 %>
