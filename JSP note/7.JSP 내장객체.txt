1. 내장 객체
    1) 웹 프로그래밍의 근간이 되는 요청과 응답부터 출력, 세션, 페이지와 어플리케이션등 제공
    2) JSP 페이지가 실행 될 때 컨테이너가 자동으로 내장 객체 생성해줌
        - 즉시 사용가능
    3) 종류
        - request : 클라이언트의 요청 정보 저장함
        - response : 클라이언트의 요청에 대한 응답 정보를 저장함
        - out : JSP 페이지에서 출력할 내용을 담는 출력 스트림
        - session : 웹 브라우저 정보를 유지하기 위한 세션 정보 저장함
        - application : 웹 애플리케이션 관련 컨텍스트 정보를 저장함
        - pageContext : JSP페이지에 대한 정보를 저장함 
        - page : JSP 페이지를 구현한 자바 클래스의 인스턴스
        - Config : JSP 페이지에 대한 설정 정보 저장함
        - exception : 예외가 발생한 경우 사용

2. request 객체 
    1) 가장 많이 사용되는 객체
    2) 클라이언트 (주로 웹 브라우저)가 전송한 요청 정보를 담고 있는 객체임
        - 클라이언트와 서버에 대한 정보 읽기
        - 클라이언트가 전송한 요청 매개변수에 대한 정보 읽기
        - 요청 헤더 및 쿠키 정보 읽기

3. response 객체
    1) 요청에 대한 응답을 웹 브라우저로 보내주는 역할을 함.
    2) 페이지 이동을 위한 리다이렉트(redirect) 
        - sendRedirect()로 페이지 이동하기

4. out 객체
    1) 웹 브라우저에 변수 등의 값을 출력할 때 주로사용.

5. application 객체
    1) 웹 어플리케이션 당 하나만 생성
    2) 모든 JSP 페이지에서 접근할 수 있음
    3) 서버의 정보, 서버의 물리적 경로 등
    4) web.xml에 설정한 컨텍스트 초기화 

6. exception 객체
    1) HTTP 에러 코드
        - 404 : NOT FOUND (클라이언트가 요청한 경로에 문서를 찾을 수 없음)
        - 405 : Method Not Allowed (GET 방식 혹은 POST방식으로 요청했는데 처리할 컨트롤러가 없는경우)
        - 500 : Internal Server Error (코드에 오타가 있거나 로직에 문제가 있어 웹 서버가 요청사항 수행 못하는 경우)
