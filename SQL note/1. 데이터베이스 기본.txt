1. 데이터베이스 정의
    1) 특정 조직의 여러 사용자가 공유하여 사용할 수 있도록 통합해서 저장한 운영 데이터의 집합
    
2. 데이터베이스 사용 이전
    1) 일반적인 텍스트 파일 형태로 저장 및 관리 되었음.
        - 파일 형태는 여러 사용자가 동시에 공유하기가 어려움.
        - 파일의 데이터의 유실 가능성이 항상 존재함.
    2) 파일 시스템 사용시 문제점
        - 데이터 중복성 문제 
        - 업데이트시 일관성(consistency) 유지가 어려움
        - 데이터 무결성(Data integrity Constraints) 유지가 어려움
        - 데이터 종속성
            - 파일구조가 바뀔때마다 응용프로그램 교체 필요
        - 동시성(Concurrency) 제공이 어려움
        - 원자성(atomicity) 제공이 어려움
            - 파일 변경중에 시스템 장애가 발생했을 때 처리가 어려움
        - 보안(Security) 제공 이슈
            - 데이터 읽기 권한 제어 이슈 가능

3. 데이터베이스 사용 이후
    - 모든 것이 데이터베이스로 관리되는 시대
    - 예시
        1) 은행
            - 계좌정보, 입출금 내역
        2) 항공사
            - 예약정보, 비행기 스케줄
        3) 대학교
            - 학생정보, 수강신청
        4) 온라인 쇼핑몰
            - 고객기록, 주문 내역
        5) 제조업
            - 제품목록, 주문, 재고, 공급망
        6) 회사
            - 직원정보, 연봉

4. 데이터베이스 특징
    1) 실시간 접근 가능
        ex) 목록 조회
    2) 계속적으로 변화
        ex) 구입정보, 물건재고 정보
    3) 동시 공유가 가능
        ex) 많은 고객이 동시 접속, 구매 가능
    4) 저장된 주소가 아닌 내용으로 참조 가능
        ex) 실시간 구매율 비교

5. 데이터의 유형
    1) 정형 데이터 (structured data) 
        - 구조화
        - 엑셀, 관계형 데이터베이스의 테이블
    2) 반정형 데이터 (semi structured data) 
        - 형태는 존재하지만, 유동적
        - HTML, XML, JSON 
    3) 비정형 데이터 (unstructured data)
        - 정해진 구조가 없이 저장된 데이터
        - text, 멀티미디어 데이터(동영상 등)
