2. 성능 데이터 모델링 정의
    - 정규화, 반정규화, 테이블통합, 테이블분할, 조인구조개선, PK, FK 등 여러가지 성능과 관련된 사항이 
      데이터 모델링 설계때부터 반영될 수 있도록 하는것.

3. 성능 데이터 모델링 고려사항
    1) 데이터 모델링을 할 때 정규화를 정확하게 수행함.  more important
        - 분산시키는 효과가 있기때문에 그 자체로 성능을 향상시키는 효과가 있음.
    2) 용량과 트랜젝션의 유형에 따라 반 정규화를 수행한다.
        - ex) 예약 화면에서 예약 완료가 안됬으나 내가 선택한 좌석을 다른사람이 볼수 없고,
              취소했을경우 다른 사람이 다시 확인 할수 있게하는것.
    3) 이력 모델의 조정, PK/FK 조정 등을 수행한다.
    4) 성능관점에서 데이터 모델을 검증한다.

4. 정규화를 통한 성능 향상 전략
    1) 결정자(주 식별자/PK)에 의해 함수적 종속을 가지고 있는 일반 속성을 의존자로 하여 CRUD를 수행한다.
    2) 한 테이블에 데이터 용량이 최소화 되는 효과가 있다.
    3) 정규화된 데이터 모델
        - 조회 ====> 처리조건 ==> 성능향상
                               성능저하                               
        - 입력,수정,삭제 ===> 성능향상

5. 정규화 용어
    1) 정규화 (Normalization)
        - 관계형 데이터 베이스 테이블의 삽입,삭제,갱신,이상현상(Anomaly) 발생을 최소화 하기위해
          좀 더 나은 단위의 테이블로 설계하는 과정
    2) 함수적 종속성 (FD : Functional Dependency)
        - 테이블의 특정 컬럼 A의 값을 알면 다른 컬럼 B의 값을 알 수 있을 때,
          컬럼 B는 컬럼 A에 함수적 종속성이 있다고 함.
          (ex. 고객명은 고객주민등록번호에 함수적 종속성이 있음.)
    3) 결정자
        - 함수적 종속성에서, 컬럼 A를 결정자라고 함.
          (ex. 주민등록번호는 결정자임.)

6. 정규화 이론
    1) 1차,2차,3차, 보이스코드 정규화는 함수적 종속성에 근거하여 정규화를 수행함.
    2) 1차 정규화 (제 1 정규형)
        - 함수적 종속성, 복수의 속성값을 갖는 속성을 분리, 속성의 원자성 확보
        - 모든 속성은 원자 값을 가져야함(한개의 값)
    2) 2차 정규화 (제 2 정규형)
        - 제 1 정규형을 만족하는 상태
        - Non-key 컬럼은 기본키 전체에 종속되어야 한다.
        - 함수적 종속성, 주 식별자에 완전 종속적이지 않은 속성의 분리(부분종속 속성 분리)
            - 기본 키에 종속적이지 않거나 기본 키 일부 컬럼에만 종속적인 컬럼은 분리되야 함.
    3) 3차 정규화 (제 3 정규형)
        - 제 2 정규형을 만족한는 상태
        - 함수적 종속성, 일반속성들 간에 종속적인 속성의 분리

7. 정규화 성능
    1) 정규화를 수행해서 조인이 발생하게 되더라도 효율적인 인덱스 사용을 통해 
       조인 연산을 수행하면 성능상 단점은 거의 없다.
