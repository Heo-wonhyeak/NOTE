1. 트랜젝션(Transaction)
    0) 더 이상 나눌수 없는 작업의 단위
    1) 여러 개의 DML 명령문을 하나의 논리적인 작업 단위로 묶어서 관리하는 것
    2) All or Nothing 방식으로 작업을 처리함으로써 작업의 일관성을 유지함
    3) 웹 애플리케이션에선 Service 클래스의 각 메서드가 애플리케이션의 단위 기능을 수행

2. Transaction의 속성 (ACID)
    1) 원자성(Atomicity) - 나눌수 없는 하나의 작업으로 다뤄져야 함
    2) 일관성(Consistency) - Tx 수행 전과 후가 일관된 상태를 유지해야 함
    3) 고립성(Isolation) - 각 Tx는 독립적으로 수행되어야 함
    4) 영속성(Durability) - 성공한 Tx의 결과는 유지되어야 함

3. 웹 애플리케이션에서 묶어서 처리하는 단위 기능 예
    1) 게시판 글 조회 시 해당 글을 조회하는 기능과 조회 수를 갱신하는 기능
    2) 쇼핑몰에서 상품 주문시 주문 상품을 테이블에 등록하는 기능과 주문자의 포인트를 갱신하는 기능
    3) 은행에서 이체시 송금자의 잔고를 갱신하는 기능과 수신자의 잔고를 갱신하는 기능
        - 출금과 입금이 모두 성공하지 않으면 실패
        - 두 계좌 잔고의 갱신이 모두 정상적으로 이루어진 후 최정적으로 반영(commit)함
            - 작업 내용을 DB에 영구적으로 저장
        - 중간에 이상이 발생할 경우 이전의 모든 작업 취소, 즉 롤백(rollback) 시킴
            - 마지막 커밋으로 복귀 (최근 변경사항을 취소)

4. 스프링의 트랜젝션 속성
    1) propergation
        - 트랜젝션 전파 규칙 설정
            - REQUIRED
                - 트랜젝션 필요
                - 진행중인 트랜젝션이 있는 경우 해당 트랜젝션에 사용
                - 트랜젝션이 없으면 새로운 트랜젝션 생성, 디폴트 값
            - REQUIRED_NEW
                - Tx이 진행 중이건 아니건, 새로 Tx 시작
    2) isolation
    3) readOnly
    4) rollbackFor
    5) noRollbackFor
    6) time
