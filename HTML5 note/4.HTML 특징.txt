1. 태그의 부모 자식 관계
    1) 웹 페이지는 '트리 구조'로 구성되어 보임.
        - 각 요소가 하나 이상의 요소에 연결되는 데이터 구조 유형

2. 검색 엔진
    1) search engine
        - 웹 사이트나 웹 페이지 등을 검색해주는 시스템이나 프로그램 등을 통칭함.
        - 웹에서 정보를 찾는 데 걸리는 시간을 최소화해 줌.
    2) 검색엔진 동작 원리
        - 결과를 빠르게 보여주려고 수많은 정보를 미리 정리해 놓음.
        - 이 과정을 자동으로 DB시키는 프로그램을 봇(bot) 또는 크롤러(crawler)라고 부름.
            - 검색 엔진은 봇이 주기적으로 웹 사이트들을 방문하여 쌓은 정보를 정리한 후, DB에 저장함.
            - 이 정보를 이후 사용자가 검색할 때 결과를 빠르고 정확하게 보여주는데 사용됨.
    3) 검색엔진의 최적화
        1] Search Engine Optimization (SEO)
        2] 검색이 잘 되도록 설계하는 일.
        3] HTML로 웹 페이지를 만들면서 검색 엔진 최적화를 해야함.
            - 메타 태그 : 검색 엔진에 정보를 제공할 목적의 태그
            - 시맨틱 태그 : 웹 페이지 구조를 구성하는 태그
                - Semantic(의미론적인)
                - 의미있는 태그
                - 검색 엔진은 HTML 코드만으로 의미를 인식해야 함.
                - 시맨틱 요소로 구성된 웹 페이지는 검색 엔진에 문서 정보를 더 의미있게 전달 할 수 있음.
                - 검색 엔진이 읽을 수 있는 문서 구조를 작성하는데 사용됨.
                    * 시맨틱 태그 종류
                    - h1, p, form, label
                    - 레이아웃 관련 태그
                      header : (웹페이지 맨위에 들어감) 소개 요소 또는 탐색 링크 나타냄
                      nav : 주로 메뉴를 구현함 
                      aside : 사이드바처럼 좌우에 위치하는 공간
                      section : 본문 내용을 포함하는 공간
                      article : 독립적인 요소 
                      footer : 문서의 하단을 정의함 (ex.저작권 명시용도)
            cf) - 비 시맨틱태그
                    - div, span

3. HTML 전역 속성
    1) title : 요소의 정보나 설명을 지정
        - <태그 title = "설명"></태그>
    2) style : 요소에 적용할 스타일(css)을 지정
        - <태그 style = "스타일"></태그>
    3) class : 요소를 지칭하는 중복 가능한 이름
        - <태그 class = "클래스이름"></태그>
    4) id : 요소를 지칭하는 고유한 이름
        - <태그 id= "고유한 아이디"></태그>

4. 글자와 상자
    1) 요소가 화면에 출력되는 특성 크게 2가지로 구분.
        - 인라인(Inline) 요소 : 글자를 만들기 위한 요소들
            - 인라인 요소 안에는 블록요소는 들어갈 수 없다!(사용불가!)
            - 인라인 요소 안에는 인라인 요소만 들어갈 수 있다.
        - 블록(Block) 요소 : 상자(레이아웃)를 만들기 위한 요소들
            - 블록요소에는 인라인요소/블록요소가 하위로 모두 들어올 수 있음
